# Safety Rules

## 確認なしにやらない

- 複数ファイルの一括削除
- `git reset --hard`, `git push --force`
- GAS のデプロイ設定変更
- スプレッドシートの構造変更（列追加/削除）
- API キーやシークレットの変更

## 確認してからやる

- 新しいファイルの作成
- 依存関係の追加
- フォルダ構造の変更
- GAS のコード変更

## 自由にやっていい

- src/ 内のソースコード編集
- CSS のスタイル調整
- JS のロジック修正
- ドキュメントの更新
- `git status`, `git diff`, `git log`
- ファイルの読み取り

## データ保護

- スプレッドシートのデータを直接削除しない
- テスト用のダミーデータを使う
- 本番データを変更する前に確認

## セキュリティ

- API キーをコードにハードコードしない
- GAS の Web アプリは適切なアクセス制限
- CORS 設定は必要最小限に
