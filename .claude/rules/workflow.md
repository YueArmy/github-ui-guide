# Workflow Rules

## 実装の流れ

1. 要件を理解
2. 既存コードのパターンを確認
3. 複雑な場合はアプローチを提案（シンプルな変更はスキップ）
4. 実装
5. ブラウザで動作確認
6. 完了報告

## 依存関係の確認

実装前に以下を確認:
- HTML ↔ CSS のクラス名の整合性
- JS ↔ HTML の要素ID/クラスの整合性
- JS ↔ GAS のAPI仕様の整合性
- GAS ↔ スプレッドシートの列構造の整合性

## エラー発生時

1. エラーメッセージを注意深く読む
2. 自分で修正を試みる
3. 3回試して解決しない場合、試したことを報告
4. 「できない」と言う前に必ず試す

## 実装完了後

報告フォーマット:
```
## 完了

**変更:**
- path/to/file.html (新規/編集/削除)

**内容:**
- 何をしたか

**確認方法:**
- ブラウザで index.html を開く
- [具体的な操作手順]
```

その後「コミットする？」と確認
